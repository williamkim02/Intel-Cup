# 배포 모델 A→Z (발표용 설명서)

> 2026 Intel Cup — AI-Based Health Screening for Second-Life Lithium-Ion Batteries
> 발표에서 직접 설명할 수 있도록 정리한 가이드. 숫자는 final report / repo 코드 기준.

## A. 문제 — 왜 이걸 만드는가
은퇴한 EV 배터리는 second-life(ESS·백업전원)로 가치가 있지만, 차에서 떼는 순간 **BMS 건강 기록이 사라진다.** 수천 개를 분류하려면 셀당 **완전 충·방전(수 시간)** 이나 **EIS 장비** 가 필요한데 너무 느리다. 폐배터리는 **이력·정격용량(명판)도 모른다.**
→ 목표: **짧은 부분 방전 + 명판 없이 SOH 추정.**

## B. 데이터 & 평가 규칙
- **NASA PCoE 18650**, 정격 2.0Ah, 2A CC 방전.
- **Leave-One-Cell-Out**: B0005/B0006/B0007 학습 → **처음 보는 B0018 검증.** 랜덤 사이클 암기가 아니라 **셀 간 전이**를 봄.
- 라벨: `SOH = Q_measured / Q_rated × 100` — **정의·라벨에만 쓰고 모델 입력엔 안 넣음.**

## C. 시행착오 (버린 방법 = "왜 이 모델인가"의 근거)
| 방법 | 결과 (B0018) | 버린 이유 |
|------|:---:|------|
| 저항 Re/Rct | R² < 0 | 절대 저항이 셀마다 달라 전이 실패 |
| 완전방전 용량 PINN | R²=0.944 | 정확하나 **cap_ratio 용량 누출**(순환), 완전방전 필요 |
| 충전곡선 PINN | 0.32 ± 0.27 | 시드마다 요동, 불안정 |
| **±10% 방전 세그먼트 PINN** ★ | **0.895 ± 0.013** | **채택** — 빠름·안정·누출 없음 |

## D. 배포 모델 — 무엇인가
**입력 = 방전곡선 10% 구간의 4개 특징 (전부 전압):**
`V_start`(구간 시작 전압) · `V_end`(끝 전압) · `ΔV = V_start−V_end` · `SOC_mid`(구간이 곡선의 어디쯤).
**용량·정격 안 들어감 → leakage-free.**

**구조:** 완전연결 신경망
`Linear(4→128) → Tanh → 128 → Tanh → 128 → Tanh → Linear(128→1) → sigmoid`
- ~33k 파라미터, **FP16 79KB**. sigmoid 출력 ×100 = SOH%.
- 학습 손실 = `MSE + λ·(SOH>50% 경계 제약)` — 이게 "physics-informed"의 실제 내용(가벼운 물리 제약).

## E. 추론 5단계
```
1. 방전 로그(time,V,I) 입력   (실측이면 먼저 디노이즈)
2. 전압곡선을 128스텝으로 리샘플 (셀마다 길이 달라도 축 통일)
3. 10% 윈도우를 5% 간격으로 슬라이딩 → ~20개  (F 참고)
4. 각 윈도우: [V_start,V_end,ΔV,SOC_mid] → 정규화 → 신경망 → 윈도우 SOH
5. 윈도우 평균 = per-cycle SOH → Good/Marginal/Replace
```

## F. 슬라이딩 윈도우 — "10% 윈도우 · 5% 간격 · ~20개"의 의미
- 방전곡선을 128점으로 편 뒤, **윈도우 = 곡선의 연속된 조각.** 10% 윈도우 = **13점**(13/128≈10%).
- **5% 간격(stride)** = 윈도우 시작점을 **6점씩**(6/128≈5%) 앞으로 이동. 이전 윈도우와 **약 50% 겹침.**
- 윈도우: `[0:13], [6:19], [12:25], …` → 시작점 0,6,12,…,114 → **총 20개.** 곡선 전체를 촘촘히 덮음.
- **각 윈도우 = SOH 하나 예측.** 20개 평균 = per-cycle SOH(안정적).
- 비유: 긴 책을 겹치는 한 페이지 스냅샷 여러 장으로 읽기 — 한 장(단일 윈도우)으로도 추정 가능하나 노이즈, 여러 장 평균이면 확신.
- **핵심:** 배포 시엔 아무 **한 조각(10%)** 만 있어도 되고(부분방전), 완전곡선이 있으면 20조각 평균으로 더 정확.

## G. SOC_mid — 유일한 "위치" 입력, 두 방식
같은 ΔV라도 곡선 위(plateau)냐 아래(knee)냐에 따라 의미가 달라 **위치 맥락** 필요.
- **방식1 (위치):** 완전방전 곡선상 위치 `1 − 중심/128`.
- **방식2 (OCV→SOC):** `voltage_to_soc((V_start+V_end)/2)` — 전압↔SOC 참조곡선(건강한 NASA 셀)을 역변환 → **부분 윈도우만으로** SOC_mid를 전압에서 추정(용량 불필요). 근사(±0.2)지만 최종 SOH는 ~1%p 안에서 유지.

## H. 집계 & 판정
- 단일 윈도우 ±몇 %p 산포 → **평균**이 안정 (per-window R²=0.895 vs per-cycle **0.921**).
- **3-class:** Good >80 / Marginal 70–80 / Replace <70. + 잔여용량(표시용).

## I. 온디바이스 배포 (Intel DK-2500)
PyTorch → ONNX → **OpenVINO IR FP16** → 엔진 선택.

| 엔진 | Latency | Power |
|------|--------:|------:|
| CPU | 0.033 ms | 22.6 W |
| iGPU | 0.062 ms | 14.8 W |
| **NPU** ★ | 0.259 ms | **7.1 W** |

셋 다 sub-ms → 지연은 병목 아님. **NPU를 전력(7.1W, CPU의 1/3)으로 선택** → 연속 계측기를 15W 안에 유지.
⚠️ 이 OpenVINO 배포/측정은 **보드 위**에서 수행했고, repo 코드는 PyTorch까지다.

## J. 검증
1. **NASA held-out B0018:** per-cycle R²=0.921, RMSE 2.17%.
2. **실제 하드웨어(DK-2500 실측):** 모델 94.0% vs 쿨롱 실측 95.9% → **오차 1.9%p, 같은 Good.** 용량·정격 모른 채 전압만으로. (`Real Data/`)
3. **임의 구간:** 90–60% 부분방전에서 73–63% 구간 하나 → 98.7%. 같은 구간이 B0018 일생 참 SOH를 **r=0.98**로 추종 → 구간이 진짜 열화를 측정. (`Real Data/figures/arbitrary_window_demo.png`)

## K. 정직한 한계 (먼저 말하면 강함)
1. **모델 천장 = 쿨롱카운팅** — 라벨이 용량 정의라, 쿨롱 못 쓰는 조건에서 그걸 근사하는 대리모델.
2. **단일 구간 절대오차 ±몇 %p** — 셀별 전압 오프셋. 상대판정(r=0.98)은 정확, 절대값은 여러 구간 평균 권장.
3. **건강한 셀 위주 실측** — 경계(70–80%) 실측은 아직.
4. **NASA 4셀** — 결론 범위 제한.

## L. 한 줄 피치
> **완전방전도 명판도 없는 폐배터리를, 방전곡선의 짧은 10% 전압 조각만으로 몇 %p 안에 SOH 판정 — 79KB 모델이 NPU에서 7.1W로 온보드 구동.**

## 예상 질문 대비
- **NPU가 왜 느려?** → 모델이 초소형이라 지연은 디스패치 오버헤드 지배. 우린 지연 아닌 **전력**으로 선택.
- **용량 알면 SOH 아는데 모델 의미?** → 완전방전+명판 가능하면 쿨롱이 정답. 우린 그게 **불가능한 조건**에서 근사.
- **아무 구간이나 되나?** → 네, r=0.98로 열화 추종. 단일 구간 ±몇 %p, 평균하면 안정.
- **physics-informed 어디?** → sigmoid 범위 보장 + SOH 경계 제약. 강한 ODE는 폐기한 저항 모델에 있었음(정직하게 "가벼운 물리 제약").
